---

## Install on Umbrel

### Method 1 — Community App Store (recommended)

1. Open your Umbrel dashboard → **App Store**
2. Scroll to the bottom → **Community App Stores**
3. Paste this URL and click **Add**:
   ```
   https://github.com/YOUR-GITHUB-USERNAME/umbrel-pulsewatch
   ```
4. Find **PulseWatch** in the store and click **Install**

### Method 2 — Manual (SSH)

```bash
# SSH into your Umbrel
ssh umbrel@umbrel.local   # password: your Umbrel password

# Install the app
~/umbrel/scripts/app install pulsewatch
```

## Android App (PWA)

PulseWatch ships a Progressive Web App installable on Android:

1. Open **Chrome** on Android
2. Visit `http://umbrel.local:4000/app`
3. Tap **Add to Home Screen** from Chrome's menu
4. PulseWatch appears on your launcher — full-screen, no browser chrome

## Features

- **HTTP & Heartbeat monitors** — poll URLs or receive push pings
- **Public status pages** — shareable at `/status/your-slug`, with RSS & Atom feeds
- **Incident management** — 3 severity levels, timestamped updates, auto-create/resolve
- **Maintenance windows** — scheduled downtime shown on status pages
- **Discord & webhook notifications** — with cooldown and HMAC signing
- **Two-factor auth** — TOTP (Google Authenticator, Authy)
- **Docker auto-restart** — link monitors to containers, auto-restart on failure
- **Server metrics** — CPU, RAM, disk usage on Analytics page
- **Zero external DB** — SQLite stored in `$APP_DATA_DIR`, survives updates

---

## Data & Privacy

All data stays on your Umbrel. PulseWatch makes no external connections except:
- Pinging the URLs you configure
- Sending notifications to webhooks you configure
- Loading fonts from Google Fonts (can be removed if needed)

The SQLite database is stored at:
```
~/umbrel/app-data/pulsewatch/data/pulsewatch.db
```

---

## Backup

The database file is stored in Umbrel's standard app data directory and is included in Umbrel's built-in backup system automatically.

To manually back up:
```bash
cp ~/umbrel/app-data/pulsewatch/data/pulsewatch.db ./pulsewatch-backup.db
```

---

## Docker integration

PulseWatch can automatically restart Docker containers when a linked monitor goes down.

The Docker socket is mounted **read-only** (`/var/run/docker.sock:ro`) — PulseWatch can only call the restart endpoint, not create/delete containers.

To link a container: **Edit monitor → Docker Auto-Restart → select container name**.

## Building the image yourself

```bash
git clone https://github.com/YOUR-GITHUB-USERNAME/umbrel-pulsewatch
cd umbrel-pulsewatch/pulsewatch

# Build for your current architecture
docker build -t pulsewatch:local .

# Build multi-arch (requires buildx + QEMU)
docker buildx build \
  --platform linux/amd64,linux/arm64 \
  -t ghcr.io/YOUR-GITHUB-USERNAME/pulsewatch:4.0.0 \
  --push .
```
